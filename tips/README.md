# Tips：テクニック集

🎯 **目標**: アルゴリズム以外の重要スキルを身につける

---

## 📁 収録内容

| フォルダ | 内容 | 問題数 |
|----------|------|--------|
| `dp_pattern/` | DPの型（配るDP / 貰うDP） | 5問 |
| `complexity_estimation/` | 制約から計算量を逆算 | 5問 |

---

## 🔑 各テクニックの概要

### dp_pattern（DPの型）

DPには「配るDP」と「貰うDP」の2つの書き方がある。

- **配るDP**: `dp[i]` から `dp[i+1]` へ値を渡す
- **貰うDP**: `dp[i]` を `dp[i-1]` などから受け取る

どちらでも書けるようになると、問題に応じて楽な方を選べる。

### complexity_estimation（計算量逆算）

制約を見て「どのアルゴリズムが使えるか」を判断する。

| 制約 | 許容計算量 | 使えるアルゴリズム例 |
|------|------------|----------------------|
| N ≤ 10 | O(N!) | 全順列 |
| N ≤ 20 | O(2^N) | bit全探索 |
| N ≤ 100 | O(N³) | ワーシャルフロイド |
| N ≤ 1000 | O(N²) | 2重ループ |
| N ≤ 10⁵ | O(N log N) | ソート、二分探索 |
| N ≤ 10⁶ | O(N) | 線形探索、累積和 |

---

## 💡 学習のコツ

1. **dp_pattern**: 同じ問題を両方の書き方で解いてみる
2. **complexity_estimation**: 問題を解く前に制約を見る癖をつける

---

## 📌 いつ使う？

- DPを書くとき → `dp_pattern/` で型を確認
- 問題を見たとき → `complexity_estimation/` で計算量を見積もる
